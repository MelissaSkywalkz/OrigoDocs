    Felsökning

    Symptom → Fix

    Metod hos oss

    1. Testa direkt mot GeoServer (GetMap/GetLegendGraphic/GetFeatureInfo).
    2. Kontrollera capabilities, REST och loggar.
    3. Isolera lager för lager, stil för stil.
    4. Felsök Origo sist.

    Legend-fel: RULE=undefined

    Symptom: Specified style does not contains a rule named undefined.

    Orsak: klienten skickar RULE utan giltigt värde eller med Rule Name som inte finns i SLD.

    Verifiera: kontrollera request-URL och matcha mot <Rule><Name> i aktuell stil.

    Fix: skicka RULE endast när värdet är giltigt och synkat med SLD Rule Name.

    Datakälla trasig: schema/flush-fel

    Symptom: Schema '...' does not exist / Unable to flush 'workspace:layer'.

    Orsak: tabell/vy saknas eller store pekar mot fel databas/schema/behörighet.

    Verifiera: öppna lager i GeoServer, testa anslutning i store, kontrollera Tomcat-loggar.

    Fix: återskapa tabell/vy, peka om store/lager eller avpublicera lagret och rensa referenser i layergroups/klient.

    Legend-fel p.g.a. datakälla

    Symptom: Can't obtain the schema for the required layer vid GetLegendGraphic.

    Orsak: samma grundorsak som schema/flush-problem.

    Fix: börja i datakällan, inte i frontend.

    Snabb checklista innan incident stängs

    • Lager svarar med korrekt statuskod i GetMap.
    • Legend fungerar utan RULE-fel.
    • Cache-läge (WMS/WMTS) verifierat för lagret.
    • Ändring dokumenterad i changelog.
