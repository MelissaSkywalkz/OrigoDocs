Felsökningshandbok

Symptom → trolig orsak → verifiera → fix

Legendfel: "Specified style does not contains a rule named undefined"

Trolig orsak: RULE-parametern skickas som undefined eller pekar på fel Rule Name.

Verifiera: kontrollera nätverksanrop för GetLegendGraphic och om RULE=undefined förekommer.

Fix: skicka RULE bara vid giltigt värde och matcha <Rule><Name> i SLD.

Fel: "Schema does not exist" / "Unable to flush"

Trolig orsak: tabell/vy saknas eller store pekar fel (schema/databas/behörighet).

Verifiera: GeoServer-logg + datakälla för lagret.

Fix: återställ datakälla, peka om store, eller avpublicera/rensa lagret i layergroups/klient.

Legendfel: "Can't obtain the schema for the required layer"

Trolig orsak: samma datakälleproblem som ovan; GeoServer kan inte läsa feature type.

Verifiera: testa WMS + GetLegendGraphic på samma lager och jämför loggmeddelanden.

Fix: samma som schema/flush-fel.

WMS är långsam (500–700 ms) jämfört med WMTS (70–200 ms)

Trolig orsak: renderkostnad, proxy/webbserver, Tomcat eller cacheflöde.

Verifiera: mät samma lager som WMS och WMTS, granska Tomcat + webbserverloggar.

Fix: seed WMTS där lämpligt, optimera style/filter, och felsök flaskhals i serverkedjan.

Okänt hos oss (behöver bekräftas)

  • Vilken komponent som oftast står för latens (GeoServer, proxy, nätverk, AV). Verifiera här: korrelerade loggar i Tomcat + webbserver/infra.
  • Exakt lista med lager som ska blockeras från legendhämtning i klient. Verifiera här: klientkonfig + supportloggar.
