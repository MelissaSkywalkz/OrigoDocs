Metodiskt arbetssätt (vår standard)

  1. Testa direkt mot GeoServer: GetMap, GetLegendGraphic, Capabilities.
  2. Kontrollera REST/loggar.
  3. Isolera lager för lager, stil för stil.
  4. Först därefter felsök Origo-konfig/UX.

Symptom: RULE=undefined i legend

Orsak: RULE-parameter felaktig eller tom.

Verifiera: nätverksspår + kontroll av Rule Name i SLD.

Fix: skicka RULE endast med giltig Name.

Symptom: Schema saknas / Unable to flush

Orsak: trasig datakälla (schema/tabell/vy/store/behörighet).

Verifiera: GeoServer-loggar + datakällans anslutning/schema.

Fix: återställ data eller peka om store; annars avpublicera/rensa lagerreferenser.

Symptom: Can't obtain schema for required layer

Orsak: samma grundorsak som schema/flush-fel.

Verifiera: jämför WMS och legendanrop för samma lager.

Fix: åtgärda datakällan först, inte UI.

Symptom: Origo upplevs långsam

Orsak: nästan alltid servernära (GeoServer/GWC/proxy/logik) före frontend.

Verifiera: mät WMS/WMTS-responstid direkt; jämför mot klientupplevelse.

Fix: flytta till WMTS där möjligt, minimera legendanrop och förenkla lagersammansättning.

Okänt hos oss (behöver bekräftas)

  • Exakta larmgränser per endpoint (WMS/WMTS/legend). Verifiera här: monitoreringskonfig/APM.
