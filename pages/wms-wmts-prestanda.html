Prestanda

WMS/WMTS-prestanda i vår klient

Syfte

Ge praktiska regler för när vi kör WMS respektive WMTS och hur Origo-inställningar påverkar serverlast.

WMS vs WMTS hos oss

  • WMTS: används via /geoserver/gwc/service/wmts för snabb rendering i stabila lager.
  • WMS: används via /geoserver/publik/wms och /geoserver/intern/wms när flexibilitet krävs.

Origo-val som driver last

  • Legendkontroll + många tematiska lager genererar många legendförfrågningar.
  • Många samtidigt synliga WMS-lager ger tung render-last per pan/zoom.
  • Djup gruppstruktur är bra för ordning men kan trigga många toggles om användarflödet är otydligt.

Prestandachecklista före release

  1. Kör snabb GetMap med liten bbox.
  2. Verifiera att WMTS-lager använder korrekt matrixset EPSG:3008.
  3. Mät nätverk: antal legendanrop vid normal användning.
  4. Bekräfta att standard-synliga lager är få och motiverade.

Varningar

  • CRS axis order: fel ordning kan ge tom karta trots HTTP 200.
  • Stora test-bboxar: maskerar flaskhalsar och gör felsökning långsammare.
  • Onödiga legendanrop: driver serverlast utan att tillföra användarvärde.
