Stilsättning (SLD/CSS/legend)

Syfte

Beskriva hur vi stilsätter lager så att legend, skala och cache blir förutsägbara i Origo.

Best practice

  • Versionera styles med suffix (exempel: byggnad_v3) innan byte av default style.
  • Definiera skalfönster tydligt per regel för att minska överritning.
  • Håll legendsymbolik synkad med skalfilter; annars uppstår missvisande legend i klient.

Legend i vår klient

Vi kör ofta med useLegendGraphics: false som default i Layermanager, vilket minskar initial last. Slå på legend selektivt på lager där användarstöd kräver det.

GetLegendGraphic-exempel:
/geoserver/ows?service=WMS&request=GetLegendGraphic&format=image/svg+xml&layer=intern:planlager&style=plan_v2&transparent=true

Styleändring + cache

  1. Verifiera först GetMap och GetLegendGraphic med liten BBOX.
  2. Truncate/seed endast berörda lager och zoomnivåer i GWC.
  3. Validera i Origo att rätt style faktiskt används (inte gammal tile).

Vanliga fel

  • Symptom: legend avviker från karta. Orsak: skalregler/filterskillnad mellan style och legendanrop.
  • Symptom: ändrad style syns inte. Orsak: kvarliggande GWC-cache.
  • Symptom: seg rendering. Orsak: för många regler utan scale-gating.

Okänt hos oss (behöver bekräftas)

  • Vår exakta policy för CSS vs SLD per team. Verifiera här: style-biblioteket i GeoServer data_dir / style-repo.
