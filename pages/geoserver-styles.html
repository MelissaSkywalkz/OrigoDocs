Stilsättning (SLD/CSS/legend/scale)

Syfte

Beskriva hur vi gör style-ändringar utan att slå ut legend, prestanda eller cache i Origo.

Best practice

  • Versionera style-namn (exempel: plan_v2, plan_v3) innan default byts.
  • Använd scale-gating i regler så att detaljrik symbolik inte renderas i små skalor.
  • För lager med thematicStyling: true ska legendsvar verifieras vid varje styleändring.

Legend i vår Origo-setup

Vi använder legendkontroll i klienten och många lager har tematiska legender. Undvik att skapa onödigt tunga legendanrop.

/geoserver/ows?service=WMS&request=GetLegendGraphic
&layer=publik:offentlig_verksamhet_grundskola_p
&format=image/svg+xml&transparent=true

Skala och fel vi ofta ser

  • Symptom: lager syns men legend upplevs fel. Orsak: style-regler med skalfilter matchar inte förväntad zoom.
  • Symptom: växlande symbolik mellan zoomnivåer. Orsak: överlappande rule-intervall.

Style + cache

  1. Kör GetMap med liten bbox.
  2. Kör GetLegendGraphic.
  3. Truncate/seed cache i begränsad bbox och nivåspann.
  4. Verifiera i Origo att WMTS och WMS inte divergerar visuellt.
