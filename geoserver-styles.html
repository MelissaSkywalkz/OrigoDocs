GeoServer

Styles: SLD, CSS, MBStyle

TODO: verifiera format-stöd

Översikt

GeoServer kan styla lager med olika format. Här är tre vanliga: SLD, CSS Extension och MBStyle. TODO: verifiera versioner och aktiverade extensions i vår installation.

SLD (Styled Layer Descriptor)

XML-baserat standardformat. Bra för formell och detaljerad styling.
  1. Ladda upp SLD i GeoServer.
  2. Koppla stilen till ett layer.
  3. Verifiera i WMS Preview.

CSS Extension

Förenklad CSS-liknande syntax (kräver extension).

/* Style för GeoServer */
* {
  polygon-fill: #FF0000;
}

TODO: verifiera om CSS Extension är aktiverad i vår GeoServer.

MBStyle (Mapbox Style)

JSON-format för styling som ofta används för vektor-tiles.

{
  "version": 8,
  "sources": {},
  "layers": []
}

TODO: verifiera stöd för MBStyle i vår GeoServer-version.

Checklistor

  1. Verifiera att rätt extension är installerad.
  2. Koppla stilen till rätt layer i GeoServer.
  3. Testa i WMS Preview innan du öppnar i Origo.

Vanliga misstag

  • Fel version på SLD → GeoServer vägrar tolka stilen.
  • CSS Extension saknas → stilen laddas inte.
  • MBStyle förväntar sig vektor-tiles men lagret är raster.

Hur Origo använder stilen

Origo får den färdiga kartbilden/utdata från GeoServer. Förändrad stil i GeoServer syns i Origo utan ändrad klientkod. TODO: verifiera eventuella cache-steg.
